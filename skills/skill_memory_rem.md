# SKILL: 三層記憶與 autoDream 做夢機制 (Memory & REM)

> 觸發條件：當討論長效記憶與資料整合時

## 三層自我修復記憶架構

```
┌──────────────────────────────────────────┐
│ Layer 1: MEMORY.md（索引層）              │
│ ├── 始終載入上下文（會話啟動時永久載入）   │
│ ├── 每條 ~150 字元，輕量級                │
│ ├── < 200 行 / ~25KB                     │
│ ├── 功能如目錄，非全書                    │
│ └── 四種記憶類型：用戶/回饋/專案/參考       │
├──────────────────────────────────────────┤
│ Layer 2: 主題檔案（詳細層）               │
│ ├── 按需拉取（grep/glob 找到後才讀取）    │
│ ├── 按專案主題分檔                        │
│ ├── 永不全部同時存在於上下文              │
│ └── 儲存於 memory/ 目錄                  │
├──────────────────────────────────────────┤
│ Layer 3: 原始紀錄（考古層）               │
│ ├── JSONL 格式對話紀錄                    │
│ ├── 永不完整重讀入上下文                  │
│ ├── 僅透過 grep 搜尋特定標識符            │
│ └── 支援會話恢復（continue/resume）       │
└──────────────────────────────────────────┘
```

## 「不信任自己的記憶」原則

```
MEMORY.md 記錄 → 視為 hint，不是 fact
    ↓
FileRead 對照驗證 → 檢查實際檔案/程式碼
    ↓
確認事實 → 才基於此事實行動
    ↓
嚴格的「成功後才寫入」紀律
```

## autoDream — 記憶整合做夢機制

模仿 REM 睡眠的四階段記憶整合流程：

### 三閘觸發機制
1. 距上次整合 ≥ 24 小時
2. 至少完成 5 個 session
3. 整合鎖定（mutex，防並行做夢）

### 四階段流程

```
┌──────────┐
│ Orient   │ 掃描記憶目錄、評估當前記憶狀態
└────┬─────┘
     ↓
┌──────────────────┐
│ Gather Signal    │ 從日誌收集近期信號、偵測過時記憶
└────┬─────────────┘
     ↓
┌──────────────────┐
│ Consolidate      │ 合併新信號、相對日期→絕對日期、消除矛盾事實
└────┬─────────────┘
     ↓
┌──────────────────┐
│ Prune & Index    │ 壓縮整理、更新 MEMORY.md
└──────────────────┘
```

## 壓縮重注入策略

壓縮後必須重新注入的內容：
- 最近 5 個存取檔案（每檔 ≤ 5K token）
- 活動計畫
- 相關 skill schema

## 記憶寫入規則

1. 若事實可從原始碼重新推導 → 永遠不儲存
2. 確認檔案寫入成功後才修改記憶索引
3. 記憶條目必須包含時間戳和來源標註
