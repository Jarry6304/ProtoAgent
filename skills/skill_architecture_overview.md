# SKILL: 全局架構與設計原則 (Architecture Overview)

> 觸發條件：當討論整體架構設計、系統依賴、設計原則時

## 全局架構：Agent Runtime，非 Chat Wrapper

### 三層架構模型

```
┌─────────────────────────────────────────────────┐
│              Presentation Layer                  │
│   React + Ink 終端 UI / ASCII 渲染引擎          │
│   (Int32Array 字元池、bitmask 樣式、游標 patch)  │
├─────────────────────────────────────────────────┤
│              Orchestration Layer                 │
│   Agent Loop / QueryEngine / Context Manager    │
│   (推理迴圈、工具路由、壓縮策略、快取經濟學)      │
├─────────────────────────────────────────────────┤
│              Capability Layer                    │
│   40+ Tools / Permission System / Memory Stack  │
│   (工具模組、四層權限、三層記憶、子智能體)        │
└─────────────────────────────────────────────────┘
```

**核心認知**：LLM 僅占程式碼庫的 ~1.6%（直接呼叫 AI 模型的部分）；其餘 98.4% 是基礎設施——上下文管理、工具編排、權限控制、終端渲染與多代理協調。

## 技術堆疊與設計意圖

| 技術選擇 | 具體實現 | 設計意圖 |
|----------|---------|---------|
| **Bun**（非 Node.js） | 執行環境 + 原生 Zig HTTP | 冷啟動速度、原生二進位層 |
| **React + Ink** | 終端 UI 框架（~140 元件） | 宣告式 UI = 狀態驅動渲染 |
| **Zod v4** | 全量 Schema 驗證 | runtime type safety |
| **tree-sitter** | Shell AST 解析 | 語義級安全檢查 |
| **TypeScript** | 全量嚴格模式 | 型別安全 + IDE 支援 |

## 十大架構原則

### 原則 1：推理與執行的嚴格分離
LLM 負責「想」，工具系統負責「做」，權限系統負責「審」。三者通過對話歷史作為唯一通訊媒介。

### 原則 2：快取經濟學驅動架構
快取讀取比標準輸入便宜 10 倍——這個單一經濟事實衍生了幾乎所有重要設計決策。

### 原則 3：自然語言即編排邏輯
多智能體的行為規範寫在 prompt 而非程式碼中，使得編排可在不部署的情況下更新。

### 原則 4：記憶是提示，不是事實
所有記憶都被視為「可能過時的提示」，行動前必須對照實際狀態驗證。

### 原則 5：Fail-Closed 安全設計
權限分類器不可用時預設拒絕。四層安全縱深。

### 原則 6：語義級安全（非字串匹配）
使用 tree-sitter AST 解析 shell 指令，確保安全檢查在語義層面運作。

### 原則 7：工具間零共享狀態
所有工具間的協作通過 LLM 推理迴圈中介，確保每個決策點都可追溯。

### 原則 8：漸進式上下文管理
從零成本 MicroCompact 到丟棄式 Truncation，多種策略形成漸進式退化管線。

### 原則 9：漸進式工具發現
重型工具延遲載入、技能分階段揭露、功能閘控透過編譯時死碼消除。

### 原則 10：斷路器保護一切
壓縮、權限分類、模型降級——每個可能失控的迴圈都有自動停止機制。

## 評估新架構時的檢查清單

- [ ] 是否遵守推理與執行分離？
- [ ] 對快取命中率的影響？
- [ ] 是否需要新的安全層？
- [ ] 是否有斷路器保護？
- [ ] 記憶寫入是否遵守「成功後才寫入」？
- [ ] 工具是否可延遲載入？
- [ ] 編排邏輯是否可通過 prompt 更新？
